# 1SubML

1SubML is an experimental ML-like programming language with a unified module and value language, structural subtyping, global type inference, higher rank polymorphic types, existential types, higher kinded types (no partial application), recursive types, worst-case polynomial time type checking and much more.

[Click here for an introduction to 1SubML.](docs/guide.md)

## Try It

[Try 1SubML in the web playground](https://storyyeller.github.io/1subml/index.html) -- edit, type-check, and run code directly in your browser.

## Quick Example

```ml
(* calculate fibonacci numbers using a recursive helper *)
let fib = (
    let rec fib_sub = fun {n; a; b} ->
        if n <= 1 then
            a
        else
            fib_sub {n=n - 1; a=a + b; b=a}
    ;
    fun n -> fib_sub {n; a=1; b=1}
);
// ints are arbitrary precision
print "fib 999 =", fib 999;

(* or calculate with an explicit loop *)
let fib = fun n -> (
    let r = {mut n; mut a=1; mut b=1};
    loop if r.n <= 1 then
            `Break r.a
        else (
            r.n <- r.n - 1;
            let old_a = r.a;
            r.a <- r.a + r.b;
            r.b <- old_a;
            `Continue ()
        )
);
print "fib 1000 =", fib 1000;

(* matching on variant types *)
let area = fun shape ->
    match shape with
    | `Circle {rad} -> rad *. rad *. 3.1415926
    | `Rect {length; height} -> length *. height
    ;

print "area =", area `Circle {rad=5.0};
print "area =", area `Rect {height=4.; length=2.5};
```

## Modules as Values

In 1SubML, modules are just records and module types are record types with existential (`type`) members. There is no separate module language — modules are ordinary values that can be passed to functions, returned from conditionals, and stored in data structures.

```ml
(* A "module type" for a counter with hidden state *)
alias Counter = {type s; new: s; increment: (s, int) -> s; get: s -> int};

(* "Modules" are just records, coerced using (:>) to hide the concrete type *)
let simple_counter = ({
    new = 0;
    increment = fun (n, inc) -> n + inc;
    get = fun n -> n;
} :> Counter);

(* A different implementation — state is a pair tracking step count *)
let step_counter = ({
    new = {val=0; steps=0};
    increment = fun ({val; steps}, inc) -> {val=val + inc; steps=steps+1};
    get = fun c -> c.val;
} :> Counter);

(* Functions that take "modules" are first-class functors *)
let count_to = fun (ctr: Counter, target) -> (
    mod C = ctr;
    let rec go = fun s ->
        if C.get s >= target then C.get s
        else go (C.increment (s, 1));
    go C.new
);

print count_to (simple_counter, 5);
print count_to (step_counter, 5);

(* Choose a module at runtime — just an ordinary if expression *)
mod C: Counter = if true then simple_counter else step_counter;

let v = C.new;
let v = C.increment(v, 1);
let v = C.increment(v, 1);
print C.get v;

(* Assign the "module" value to a new variable *)
let foo = C;
let v = foo.increment(v, 1);
print foo.get v;
```

## Getting Started

Build the compiler from source (you will need to have `lalrpop` installed):

```sh
lalrpop compiler_lib/src/grammar.lalr 
cargo build --release 
```

You can then invoke the compiler like this:

```sh
./target/release/cli myfile.ml 
```

The `cli` binary will automatically compile your 1SubML code to Javascript and then execute it using `node`.


To build the [web demo](https://storyyeller.github.io/1subml/index.html) from source, run `site/build.sh`. You will need to have `npm` and `wasm-pack` installed.

## AI use

The 1SubML compiler code was 99% hand-written (though with Copilot-powered autocomplete to speed up typing.) I occasionally used Claude Code to make minor changes to the compiler, but when I tried to generate larger amounts of code with Claude, I always ended up having to rewrite it all myself anyway. 

The test runner code (test_runner.rs and js_executor.rs) was generated by Claude Code with only minor edits. I also used Claude to generate most of the UI code for the web demo and some of the test files.

I wrote the [language guide](docs/guide.md) (chapters 1-4) by hand, while the table of contents and [standard library documentation](docs/stdlib.md) were generated by Claude Code (based on the library code, which I wrote by hand).
